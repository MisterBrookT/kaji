Kaji Harness Pool

把 Claude Code、Codex、MiniMax/Ark、Gemini CLI、OpenCode 等 harness 先纳入一个本地额度与会话控制面。

Mode A · one-page briefing local-first quota before routing 2026-06-17

Core decision

当前最稳的切入点不是立刻做请求路由，而是把 Kaji Gauge 扩成 harness pool cockpit：统一看额度、会话、当前账号/配置、可切换能力和重启要求。

WHY THIS FIRST

路由要处理 streaming、tool calls、模型映射、失败重试、协议转换和账号风险； quota cockpit 则复用现有 Resources/quota.py，先解决每天真实痛点： 哪个 harness 还能用、哪个快烧完、该开哪个新 session。

建议严格按层推进。Observe 层不改用户配置，风险最低；Route 层最后做，且应是独立 daemon。

Proposed phases

1

Harness inventory

扩展 quota.py 输出结构：harness id、quota、sessions、cwd、capabilities。仍然只读。
2

Profile registry

本地记录 provider profile、model alias、默认 harness binding；先展示，不写入目标工具配置。
3

Safe config switching

挑一个 harness 做原子写入、备份、验证、rollback，并明确标注 needs restart。
4

Optional local router

独立 localhost daemon；按协议边界做 adapter，不做一个万能 endpoint。
